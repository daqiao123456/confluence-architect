# Confluence 架构师评分标准

供自测和质量验收使用。评分分两层：硬门（pass/fail）+ 加权维度（1-5分）。

---

## 一、硬门（全过才算通过）

任一硬门未通过，整体判定为 **不通过**，无需计算加权分。

### 通用硬门（所有档位）

| # | 检查项 | 判定标准 |
|:---:|:---|:---|
| G1 | 档位标注 | 方案开头是否明确标注档位代号 |
| G2 | 模块完整性 | 输出模块是否与 `ref.md` 第八节模块矩阵一致（✅/⚠️ 的都有，❌ 的都没有） |
| G3 | 成树层数 | 是否符合档位要求（Ⅰ=2层为主、Ⅱ=3层、Ⅲ+=4层） |
| G4 | 文档类型 | 是否符合档位要求（Ⅰ=3类、Ⅱ=4类、Ⅲ+=4+3类） |
| G5 | 升降档建议 | 是否包含反模式 + 升档信号 + 迁移原则 |
| G6 | 方法论一致 | 输出是否符合 `Target/Description.md` 的"道法术器治理度量"体系 |
| G7 | 规模适配 | 输出是否符合 `Target/Scale.md` 的档位规则，不过度不缺位 |

### Ⅲ档及以上专项硬门

| # | 检查项 | 判定标准 |
|:---:|:---|:---|
| H1 | 列事-列文对应 | 列文矩阵行数 = 列事对象组数，一个不漏 |
| H2 | 列事-列位对应 | 列位表覆盖所有对象组 + 容器 |
| H3 | 列事-成树对应 | 一级目录数 = 大类数，一一对应 |
| H4 | 成树展开完整 | 每个大类展开到对象组层和文档类型层（第3层） |

---

## 二、加权维度（1-5分）

硬门全通过后，按以下 4 个维度打分。

### 维度定义

| 维度 | 权重 | 衡量什么 |
|:---|:---:|:---|
| 完整性 | 35% | 各模块内容是否充实，不是只有标题 |
| 准确性 | 25% | 分类法、映射关系是否与 ref.md 数据一致 |
| 档位适配 | 25% | 每个模块的深度是否严格匹配当前档位 |
| 可操作性 | 15% | 输出能否直接用于 Confluence 建站 |

### 锚点定义

| 分数 | 完整性 | 准确性 | 档位适配 | 可操作性 |
|:---:|:---|:---|:---|:---|
| 5 | 各模块内容充实，模块间相互引用 | 完全匹配分类法 + 行业适配 | 每个模块严格匹配档位深度 | 可直接建站，无需二次翻译 |
| 4 | 模块齐全，个别内容可更丰富 | 基本匹配，1处微小偏差 | 基本匹配，1处微小越界 | 大部分可直接用，少量需调整 |
| 3 | 模块齐全但内容薄 | 基本符合但有遗漏 | 档位大致对但细节越界 | 有结构但缺执行细节 |
| 2 | 缺少次要内容 | 多处与分类法矛盾 | 明显越界或不足 | 需大量补充才能落地 |
| 1 | 缺失关键模块 | 严重偏离分类法 | 小企业输出大厂方案 | 纯理论无法落地 |

### 计算方式

```
总分 = 完整性×0.35 + 准确性×0.25 + 档位适配×0.25 + 可操作性×0.15
```

### 通过阈值

| 等级 | 总分范围 | 判定 |
|:---|:---|:---|
| 优秀 | ≥ 4.5 | 通过，可直接交付 |
| 良好 | 4.0 – 4.4 | 通过，建议微调 |
| 及格 | 3.5 – 3.9 | 有条件通过，需修正后交付 |
| 不及格 | < 3.5 | 不通过，需重新生成 |

---

## 三、按档位差异化评分要点

| 档位 | 重点检查 |
|:---|:---|
| Ⅰ 轻库 | 是否足够简单（只3类页、2层目录、无标签无度量）；语言是否通俗 |
| Ⅱ 标准 | 简化矩阵是否标了有/缺/暂缓/不需要；标签是否只3类 |
| Ⅲ 体系 | 10模块是否齐全；治理是否轻量（非完整版）；度量是否1-2项 |
| Ⅳ 联邦 | 治理是否完整五维度；是否提及中央治理主体 |
| Ⅴ 平台 | 是否有行业特化；度量四指标是否有具体衡量方式和基线值 |
